# Juya News Card 使用流程

## 概述

本项目支持两种方式生成卡片：

1. **本地 LLM**（Ollama）- 适合简单内容，可能超时
2. **线上 AI**（推荐）- 适合长文本，稳定可靠

> **提示**：本地 Ollama 因算力限制，处理长文本容易超时。推荐将提示词粘贴到线上 AI 使用。

***

## 方式一：本地 Ollama（简单内容）

### 1. 配置本地模型

编辑 `.env` 文件：

```env
LLM_API_KEY=your-api-key
LLM_API_BASE_URL=http://localhost:11434/v1
LLM_MODEL=qwen2.5:7b
```

### 2. 启动 Ollama

```bash
ollama run qwen2.5:7b
```

### 3. 输入内容生成卡片

在网页输入框粘贴内容，点击生成。

⚠️ **注意**：长文本（>2000字）容易超时失败。

***

## 方式二：线上 AI（推荐，适合长文本）

### 步骤 1：获取系统提示词

系统提示词位于：

- 文件：`网页端AI提示词.md`
- 或网页界面点击 "提示词" 按钮复制

### 步骤 2：准备你的内容

准备好需要总结的文章或新闻内容。

### 步骤 3：粘贴到线上 AI

打开任意线上 AI（推荐）：

- [ChatGPT](https://chat.openai.com)
- [Claude](https://claude.ai)
- [Gemini](https://gemini.google.com)
- [通义千问](https://tongyi.aliyun.com)
- [DeepSeek](https://deepseek.com)

**操作步骤**：

1. **第一步**：粘贴系统提示词
   ```
   [粘贴 llm-prompt.ts 中的内容]
   ```
2. **第二步**：粘贴你的文章
   ```
   请根据以上内容，生成新闻卡片 JSON 数据。

   文章内容：
   [你的文章内容]
   ```
3. **第三步**：要求 AI 输出 JSON 格式
   ```
   请直接输出 JSON，不要包含 Markdown 代码块标记。
   ```

### 步骤 4：获取 JSON 输出

AI 会返回如下格式的 JSON：

```json
{
  "mainTitle": "文章主标题",
  "cards": [
    {
      "title": "要点1",
      "desc": "详细描述...",
      "icon": "article"
    },
    {
      "title": "要点2",
      "desc": "详细描述...",
      "icon": "info"
    }
  ]
}
```

### 步骤 5：导入项目生成卡片

**方法一：直接粘贴 JSON**

1. 打开网页界面
2. 切换到 "JSON 输入" 模式
3. 粘贴 AI 生成的 JSON
4. 点击 "生成卡片"

**方法二：保存为文件导入**

1. 将 JSON 保存为文件（如 `content.json`）
2. 使用项目脚本生成：
   ```bash
   npm run generate -- --input content.json --output output.html
   ```

## 常见问题

### Q: 本地 Ollama 为什么总是超时？

A: 本地模型算力有限，处理复杂提示词需要很长时间。建议：

- 缩短输入文本
- 使用更大的模型（如 qwen2.5:14b）
- 改用线上 AI

### Q: 线上 AI 生成的 JSON 格式不对？

A: 请确保：

1. 完整粘贴系统提示词
2. 明确要求 "输出 JSON 格式"
3. 检查 AI 输出是否包含 Markdown 代码块（\`\`\`json），需要去掉

### Q: 支持哪些图标？

A: 支持 Material Design Icons，常用图标：

- `article` - 文章
- `info` - 信息
- `lightbulb` - 灵感
- `trending_up` - 趋势
- `warning` - 警告
- `check_circle` - 确认

完整列表：[Material Icons](https://fonts.google.com/icons)

## 推荐工作流

```
长文章 → 线上 AI（ChatGPT/Claude等） → JSON 输出 → 粘贴到项目 → 生成卡片
```

**优势**：

- ✅ 不依赖本地算力
- ✅ 处理速度快
- ✅ 支持超长文本
- ✅ 结果更稳定
