# Juya News Card 部署指南

## 项目简介

Juya News Card 是一个轻量级的 AI 新闻卡片生成工具，基于 Next.js 一体化架构（前后端同服）。

> **定位**: "可用优先"的普通开发者项目，适合快速分享思路与工具，不要求重度维护。

---

## 环境要求

- **Node.js**: >= 20.0.0
- **npm**: >= 10.0.0
- **内存**: 至少 2GB RAM
- **磁盘**: 至少 2GB 可用空间

---

## 快速开始（推荐）

### 方式一：Node.js 直接运行（最简单）

#### 1. 安装依赖

```bash
npm install
```

#### 2. 准备环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件，至少配置以下 3 项：

```env
LLM_API_KEY=your-api-key
LLM_API_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
```

#### 3. 安装 Playwright（首次运行必需）

```bash
npx playwright install
```

#### 4. 启动服务

```bash
# 开发模式（热重载）
npm run dev

# 或生产模式
npm run build
npm start
```

访问 http://localhost:3000 即可使用。

---

### 方式二：Docker 部署（推荐用于分享）

#### 1. 准备环境变量

```bash
cp .env.example .env
# 编辑 .env 配置 LLM API 信息
```

#### 2. 使用 Docker Compose 启动

```bash
docker compose up --build
```

访问 http://localhost:3000 即可使用。

#### 3. 常用命令

```bash
# 后台运行
docker compose up -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down

# 重新构建
docker compose up --build
```

## 环境变量配置

### 必需配置

```env
LLM_API_KEY=your-api-key-here
LLM_API_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
```

### 常用可选配置

```env
# 服务端口（默认 3000）
PORT=3000

# LLM 请求超时（默认 120000ms）
LLM_TIMEOUT_MS=120000

# 最大输入字符数（默认 12000）
MAX_INPUT_TEXT_CHARS=12000

# API 认证令牌（生产环境建议设置）
API_BEARER_TOKEN=your-random-token

# 允许未认证写入（开发环境 true，生产环境 false）
ALLOW_UNAUTHENTICATED_WRITE=true
```

## 生产环境部署

### 使用 PM2 进程管理

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start npm --name "juya-news-card" -- start

# 保存配置
pm2 save
pm2 startup

# 常用命令
pm2 status
pm2 logs juya-news-card
pm2 restart juya-news-card
pm2 stop juya-news-card
```

### Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## 故障排查

### 1. Playwright 安装失败

```bash
# 手动安装 Chromium
npx playwright install chromium

# 或安装所有浏览器
npx playwright install
```

### 2. 端口被占用

```bash
# 查找占用 3000 端口的进程
lsof -i :3000

# 或使用其他端口
PORT=3001 npm run dev
```

### 3. LLM API 连接失败

- 检查 `LLM_API_KEY` 是否正确
- 检查 `LLM_API_BASE_URL` 是否可访问
- 检查网络连接和防火墙设置

### 4. 内存不足

```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" npm run build
```
